# EchoFist 边界约束文档

## 1. 技术边界约束

### 1.1 编程语言与框架
- **核心语言**: Python 3.10+
- **禁止语言**: 不得使用 C/C++ 扩展（除非性能关键且经过评审）
- **异步框架**: 使用 asyncio 作为异步框架
- **禁止框架**: 不得使用 Tornado、Twisted 等其他异步框架
- **Web 框架**: 不得引入任何 Web 框架（如 Flask、FastAPI）

### 1.2 数据库与存储
- **主数据库**: SQLite（默认）
- **可选数据库**: PostgreSQL（仅作为可选配置）
- **禁止数据库**: 不得使用 MongoDB、Redis 等 NoSQL 数据库
- **文件存储**: 仅支持本地文件系统存储
- **云存储**: 不得集成任何云存储服务

### 1.3 网络通信
- **支持协议**: 
  - KiwiSDR WebSocket 协议
  - HTTP/HTTPS（仅用于配置获取）
- **禁止协议**: 
  - 不得实现 FTP、SFTP 等文件传输协议
  - 不得实现自定义二进制协议
- **端口范围**: 仅使用 8000-9000 范围内的端口
- **并发连接**: 最大并发连接数 ≤ 10

### 1.4 音频处理
- **采样率**: 支持 8000Hz、16000Hz、44100Hz、48000Hz
- **声道**: 仅支持单声道（mono）
- **音频格式**: 支持 WAV、RAW PCM
- **禁止格式**: 不得支持 MP3、AAC 等压缩格式
- **实时处理**: 缓冲区大小 ≤ 4096 样本

## 2. 功能边界约束

### 2.1 核心功能范围
- **必须实现**:
  1. CW 信号盲解调
  2. KiwiSDR 网络接入
  3. 手法特征提取
  4. 自动化 QSO 流程
  5. 拟人化信号重放
- **禁止实现**:
  1. 语音识别与合成
  2. 图像处理与识别
  3. 视频流处理
  4. 其他数字模式（如 RTTY、PSK31）

### 2.2 用户界面约束
- **界面类型**: 仅支持文本用户界面（TUI）
- **界面库**: 使用 Rich 库构建 TUI
- **禁止图形界面**: 不得使用 PyQt、Tkinter 等 GUI 框架
- **交互方式**: 仅支持键盘交互，不支持鼠标
- **屏幕分辨率**: 支持最小 80×24 字符终端

### 2.3 配置管理约束
- **配置文件格式**: 支持 TOML、YAML、JSON
- **配置文件位置**: 仅支持以下位置:
  - 用户主目录: `~/.echofist/config.toml`
  - 当前目录: `./echofist.toml`
- **环境变量**: 支持有限的环境变量覆盖
- **禁止远程配置**: 不得从远程服务器加载配置

## 3. 性能边界约束

### 3.1 实时性能指标
| 指标 | 最小值 | 目标值 | 最大值 | 单位 |
|------|--------|--------|--------|------|
| 音频输入延迟 | 10 | 20 | 50 | ms |
| 信号处理延迟 | 20 | 40 | 100 | ms |
| 解码延迟 | 5 | 10 | 20 | ms |
| 界面刷新延迟 | 50 | 100 | 200 | ms |
| 总端到端延迟 | 100 | 200 | 500 | ms |

### 3.2 资源使用限制
| 资源类型 | 启动时 | 运行时（峰值） | 单位 |
|----------|--------|----------------|------|
| 内存使用 | ≤ 50 | ≤ 256 | MB |
| CPU 使用率 | ≤ 5% | ≤ 80% | % |
| 磁盘空间 | ≤ 10 | ≤ 100 | MB |
| 网络带宽 | ≤ 10 | ≤ 100 | KB/s |

### 3.3 容量限制
| 项目 | 限制值 | 说明 |
|------|--------|------|
| 同时解码信号数 | 1 | 不支持多信号并行解码 |
| 历史记录保存 | 1000 | 最大保存 QSO 记录数 |
| 特征库大小 | 100 | 最大保存手法特征数 |
| 配置文件大小 | 10 | 最大配置文件大小（KB） |

## 4. 兼容性边界约束

### 4.1 操作系统兼容性
- **必须支持**:
  - Linux (Ubuntu 20.04+, CentOS 8+)
  - macOS (10.15+)
  - Windows (10+)
- **可选支持**:
  - FreeBSD
  - Raspberry Pi OS
- **禁止支持**:
  - Android
  - iOS
  - 嵌入式系统

### 4.2 Python 版本兼容性
| Python 版本 | 支持状态 | 说明 |
|-------------|----------|------|
| 3.10 | 必须支持 | 最低要求版本 |
| 3.11 | 必须支持 | 推荐版本 |
| 3.12 | 必须支持 | 最新稳定版 |
| 3.13+ | 可选支持 | 未来版本 |
| < 3.10 | 禁止支持 | 不兼容 |

### 4.3 硬件兼容性
- **必须支持**:
  - 标准声卡（支持 44.1kHz/48kHz）
  - 普通网络适配器
  - 2GB+ RAM
  - 双核 CPU
- **可选支持**:
  - USB 声卡
  - 专业音频接口
- **禁止要求**:
  - GPU 加速
  - 特殊 DSP 硬件
  - 专用无线电设备

## 5. 安全边界约束

### 5.1 数据安全
- **加密要求**:
  - 配置文件不得包含明文密码
  - 网络通信使用 TLS 1.2+
  - 敏感数据内存加密
- **数据隔离**:
  - 用户数据与系统数据分离
  - 不同用户数据隔离
  - 临时文件安全清理

### 5.2 访问控制
- **权限要求**:
  - 仅需要普通用户权限
  - 不需要 root/管理员权限
  - 文件权限限制为 644（文件）、755（目录）
- **认证机制**:
  - 支持简单的 API 密钥认证
  - 不支持复杂的 OAuth/SAML
  - 不存储用户密码

### 5.3 安全审计
- **日志要求**:
  - 记录所有配置变更
  - 记录网络连接事件
  - 记录异常访问尝试
- **监控要求**:
  - 监控异常资源使用
  - 监控网络流量异常
  - 监控文件系统访问

## 6. 法律与合规边界

### 6.1 无线电法规合规
- **频率限制**:
  - 仅支持业余无线电频段
  - 遵守当地频率分配规定
  - 支持频率范围: 1.8MHz - 30MHz
- **功率限制**:
  - 软件不控制发射功率
  - 用户需自行确保合规
  - 提供法规提示信息

### 6.2 数据隐私合规
- **数据收集**:
  - 仅收集必要的技术数据
  - 不收集个人身份信息
  - 用户可选择退出数据收集
- **数据使用**:
  - 仅用于软件功能改进
  - 不用于商业广告
  - 不共享给第三方

### 6.3 开源许可证合规
- **许可证要求**:
  - 核心代码使用 MIT 许可证
  - 依赖库必须兼容 MIT
  - 不得使用 GPL 等传染性许可证
- **版权声明**:
  - 所有代码文件包含版权声明
  - 第三方库保留原始版权
  - 贡献者签署 CLA

## 7. 架构边界约束

### 7.1 模块化约束
- **核心模块**:
  - `audio_processor`: 音频处理，依赖 numpy、scipy
  - `morse_decoder`: 摩尔斯解码，纯 Python 实现
  - `kiwi_client`: KiwiSDR 客户端，依赖 websockets
  - `fist_extractor`: 特征提取，可选依赖 torch
- **模块依赖**:
  - 禁止循环依赖
  - 依赖方向从上到下
  - 接口定义清晰

### 7.2 接口约束
- **内部接口**:
  - 使用 Python 类型注解
  - 异步接口使用 async/await
  - 错误处理使用异常
- **外部接口**:
  - 命令行接口使用 Click
  - 配置文件接口使用 Pydantic
  - 网络接口使用标准协议

### 7.3 扩展性约束
- **插件系统**: 不支持动态插件加载
- **扩展点**: 仅支持有限的配置扩展
- **API 稳定性**: 公共 API 保持向后兼容
- **废弃策略**: 提前一个版本通知废弃
